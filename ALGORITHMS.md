## машины

* поля
- общее количество памяти
- общее количество процессора

* удалениние по id
- попытка разместить процессы на других машинах, если процесс не удается разместить - он удаляется
- возврат списка удаленных процессов

## процессы
- требуемое количество памяти
- требуемое количество процессора
- id машины 

---

## общий алгоритм ребалансировки при добавлении/удалении процесса

1) для всех машин вычисляем нагрузку (load_i) по формуле:

    load_i = max(
        (used_memory)/total_memory,
        (used_cpu)/total_cpu) 
    и отсортировать в порядке возрастания

    mx - машина с max(loads)
    mn - машина с min(loads)

* цикл (до тех пор как не найдется mn такая, что выбранный процесс с mx помещается на mn
или пока mn != mx):

2) берем новый mn
3) на mx выбираем процесс для перемещения (см. ниже) 
4) если нашелся такой процесс - пытаемся выполнить перемещение
5) если получилось выполнить перемещение - break

---

### выбор процесса для ребалансировки

1) циклом проходимся по всем процессам на машине
2) ищем самый тяжелый процесс, который поместиться на целевой машине и нагрузка на целевой машине не станет больше, чем на исходной

---

### критерии остановки

прекращаем цикл ребалансировки, если:
- trashold < min_load/max_load (trashold = 0.9 по умолчанию, этот параметр можно поменять в src/Config/BalancerThreshold.php)
- за одну итерацию цикла не удалось найти процесс, который можно было бы перенести
- количество итераций превысило количество процессов в бд

---

## алгоритм ребалансировки при удалении машины

1) сортируем процессы с удаляемой машины через суммы их ресурсов в порядке убывания 
2) для всех машин вычисляем нагрузку (load_i) по формуле: load_i = max(
                   (used_memory)/total_memory,
                   (used_cpu)/total_cpu) и сортируем в порядке возрастания 
3) для каждого процесса циклом перебираем машины и если машина подходит (хватает ресурсов) - запоминаем этот процесс для нее, обновляем load_i и снова сортируем
4) если для каждого процесса нашлась новая машина - обновляем бд 
5) иначе - возвращаем ошибку с перечислением процессов (id, cpu, memory), для которых не нашлась новая машина
